Caesar Cipher

Also called a shift cipher, given a shift key, all the letters in the input text get "shifted" based on the key, for instance if the shift is 2, 'A' is now 'C', 'B' is now 'D' and so on.

Affine Cipher

A monoalphabetic substitution cipher, each letter is mapped to it's equivalent numerical value (ex. A = 1). Encryption follows the formula of (Alpha X [current letter]) + Beta mod 26. While decryption follows [Modular Multiplicative Inverse of Alpha] X ([current letter] - beta) mod 26. Caesar cipher could be referred as an Affine cipher with alpha value of 1. Since it is considered a monoalphabetic substitution cipher, it still inherits the weaknesses and can be easily breached using frequency analysis, brute force, or simply guessing.

Vigenere Cipher

This cipher uses polyalphabetic substitution, it utilizes a key made out of letters and follow a similar concept to Caesar Cipher. The key repeats itself along the plaintext and uses the numerical value of the keyword to shift the aligned plain text. The vice versa is done to a cipher text to decrypt it. For example, if we have plaintext "thisisatest" (this is a test) and the key is "key" the key would look like keykeykeyke and the ciphertext would be DLGCMQKXCCX
